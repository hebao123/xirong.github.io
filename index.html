Mac下数据备份策略

发表于 2015-12-12   |   分类于 mac   |  

事情如果有变坏的可能，不管这种可能性多么小，它总会发生（Anything that can go wrong will go wrong）。 - 墨菲定律（Murphy’s law)

在此之前你有想过备份数据的事情吗？ 无论有或者没有，从此刻起，你都应该好好思考怎么去保护自己数据的可用性、安全性，以避免数据丢失、数据泄露所带来的时间经济等各种损失。我想能看到这篇文章的你，对 mac 或者 pc 的使用上基本不会有问题，无论你从事什么职业，工作生活中少不了与电脑打交道，那么自然的你会有很多数据，写的代码，安装的各种软件，系统的配置，处理的 word 文档、excel 表格、ppt 演讲稿等等，使用电脑的时间越长，这样子的数据就越多，产生的价值也就越大，尤其是随着你的身份的变化，这些数据的影响越来越重要。以前我一直侥幸的认为，在我身上不会发生数据丢失的问题，像硬盘损坏这种概率极低的事件更不可能发生，可万万没想到，上周升级了mac 上的 iTunes ，升级过程中重启导致再也无法开机，尝试各种办法无果，最终还是去的苹果零售店维修，万幸的是数据没有丢失。由于之前没有怎么关注，许多最近的数据都没有备份，维修工程师一直强调数据很可能丢失，即使找专业的恢复团队，花费一笔不小的费用不说，还不定能全部找回，当时我滴个小心脏啊……

自此以后，把数据备份的事情重视起来，慢慢的在mac osx系统上形成了一套安全可行的方法，这里写出来供大家参考。

整体的原则是关系到系统的，比如安装的软件、bash/git/vim/zsh等各种配置，以及一些重要的工作文档，统统交给 mac 上的好助手 “time machine”，并且选择加密备份的方式；除此之外，对重要数据做到冗余备份（一份数据有多个不同的copy），存储到云服务提供商中，比如国外的icloud、drop box、google drive，国内的百度云、腾讯微云、金山快盘等。

Time machine 是我个人认为 osx 好用于 windows 的一个地方，如果使用time machine备份了自己的mac，即使硬盘坏掉、不幸被盗又或者不为人所抗拒的因素导致数据丢失了，重启机器，选择从 time machine中备份，即可恢复到你之前的工作状态，包括所有的机器设置、开机项、浏览的标签页、处理的文档、看的美剧进度统统跟之前一模一样，你只需要享受这么方便快捷的服务即可，这是很令人感动的；time machine还可以让文档有了历史版本，就像大学的论文，它可以演绎整个文件的变化过程，可以很方便的回到某个时间点，而不用担心内容的丢失。

time machine支持的可选择功能很多，备份硬盘空间不够或者不想备份某些文件，可以选择排查文件/文件夹；备份的硬盘容量1T多，总不能全部用来备份吧，可以选择将备份盘进行分区，分区完后，一部分用来备份，另一部分正常使用存储文件就好了；备份的资料被人拿到后不是都可以看到了么，选择加密备份即可解决这个问题。

当然，time machine 主要的作用还是对系统的数据和系统状态的保全，为了安全还需要综合考虑其他的备份和保全手段以及养成文档集中的习惯，比如我还选择使用网盘来冗余备份自己的重要文档。国外的dropbox、google drive，以及国内的百度云、腾讯微云、金山快盘等，都是非常不错的选择，不过也有不同点：国外的网盘对安全性、个人数据隐私上做的相对来说比国内的要好一些，至少不会检测你文件的内容是什么，从而有选择的屏蔽，缺点就是需要翻墙，对普通用户有一定的使用成本；国内的几大网盘容量超大，全终端使用基本零成本，同步速度也较快，缺点是毕竟受政治的影响，个人数据的隐私保护上是有一定差距的。无论使用哪个，最总要的是使用网盘的初衷，冗余备份数据，多一份数据安全。（通过云盘同步资料的方式在windows上也是可以使用的，win上面也可以选择此方式来备份重要的资料）

嘿，屏幕前的你，对数据备份有什么好的方式或者建议，欢迎给我留言，送你关于time machine 的使用技巧。

发表于 2015-08-22   |   分类于 java   |  

此文来自书籍《深入理解Java虚拟机-Jvm高级特性与最佳实践》的第五部分：高效并发，详细 xmind 读书笔记参考读书笔记。

volatile 修饰符的理解

关键字 volatile 可以是 Java 虚拟机提供的轻量级同步机制，但并不容易被正确、完整的理解，不知道如何、什么场景该使用，以至于在处理多线程数据竞争问题的时候一律用synchronized同步，损失了一些性能。
当一个变量被定义为volatile后，将具备两种特性：一是保证此变量对所有线程的可见性，二是禁止指令重排序优化。

可见性

这里的“可见性”是指当一条线程修改了变量的值，新值对于其他线程来说可以立即知道，而普通变量做不到。（Java内存模型是通过在变量修改后同步回住内存，在变量读取前从主内存刷新变量值这种依赖主内存作为传递媒介的方式来实现可见性，普通变量和volatile的区别是：volatile的特殊规则保证了新的值能立即同步到主内存，以及每次使用前立即从主内存刷新。）因此，volatile保证了多线程操作变量时的可见性，普通变量不能。
